# 微信公众号自定义菜单管理功能设计方案

## 1. 项目概述

### 1.1 背景
实现微信公众号自定义菜单的创建、查询、删除等管理功能，通过 Java SDK 封装微信开放平台的自定义菜单 API，简化菜单配置和管理流程。

### 1.2 功能范围
- **菜单创建**：创建自定义菜单（含一级菜单和二级菜单）
- **菜单查询**：获取当前菜单配置信息
- **菜单删除**：删除自定义菜单
- **菜单按钮类型**：支持 click、view、miniprogram、扫码、拍照等多种类型

### 1.3 参考文档
- [微信开放平台 - 创建自定义菜单](https://developers.weixin.qq.com/doc/subscription/api/custommenu/api_createcustommenu.html)
- [微信开放平台 - 自定义菜单管理](https://developers.weixin.qq.com/doc/offiaccount/Custom_Menus/Creating_Custom-Defined_Menu.html)

### 3.2 API 接口设计

#### 3.2.1 菜单管理 API (MenuApi)

```java
/**
 * 菜单管理接口
 */
public interface MenuApi extends WechatApi {

    /**
     * 创建自定义菜单
     *
     * @param request 菜单创建请求
     * @return 创建结果
     * @throws WechatApiException API 调用异常
     */
    MenuCreateResponse createMenu(MenuCreateRequest request);

    /**
     * 获取自定义菜单配置
     *
     * @return 菜单配置信息
     * @throws WechatApiException API 调用异常
     */
    MenuGetResponse getMenu();

    /**
     * 删除自定义菜单
     *
     * @return 删除结果
     * @throws WechatApiException API 调用异常
     */
    MenuDeleteResponse deleteMenu();
}
```

#### 3.3.2 创建菜单请求

**MenuCreateRequest**
```java
public class MenuCreateRequest {

    /**
     * 一级菜单数组（最多 3 个）
     */
    private List<MenuButton> button;

    public static class Builder {
        private final List<MenuButton> buttons = new ArrayList<>();

        /**
         * 添加一级菜单
         */
        public Builder addButton(MenuButton button) {
            this.buttons.add(button);
            return this;
        }

        /**
         * 添加一级菜单（使用构建器）
         */
        public Builder addButton(ButtonBuilder buttonBuilder) {
            this.buttons.add(buttonBuilder.build());
            return this;
        }

        /**
         * 构建请求对象
         */
        public MenuCreateRequest build() {
            MenuCreateRequest request = new MenuCreateRequest();
            request.button = this.buttons;
            return request;
        }
    }
}
```

## 4. 使用示例

### 4.1 快速开始

```java
import com.shazam.wechat.sdk.WechatClient;
import com.shazam.wechat.sdk.WechatConfig;
import com.shazam.wechat.sdk.model.request.*;
import com.shazam.wechat.sdk.model.response.*;

public class WechatMenuExample {
    public static void main(String[] args) {
        // 1. 创建客户端
        WechatConfig config = new WechatConfig.Builder()
            .appId("your_app_id")
            .appSecret("your_app_secret")
            .build();
        WechatClient client = new WechatClient(config);

        try {
            // 2. 获取 access_token
            client.auth().getStableAccessToken();

            // 3. 创建菜单
            MenuCreateRequest request = new MenuCreateRequest.Builder()
                .addButton(new ButtonBuilder()
                    .name("菜单 1")
                    .asClick("MENU_1_CLICK"))
                .addButton(new ButtonBuilder()
                    .name("菜单 2")
                    .asView("https://example.com"))
                .addButton(new ButtonBuilder()
                    .name("菜单 3")
                    .addSubButton(new MenuButton.Builder()
                        .name("子菜单 1")
                        .asClick("SUB_1_CLICK")
                        .build())
                    .addSubButton(new MenuButton.Builder()
                        .name("子菜单 2")
                        .asView("https://example.com/sub")
                        .build()))
                .build();

            MenuCreateResponse response = client.menu().createMenu(request);
            if (response.isSuccess()) {
                System.out.println("菜单创建成功！");
            }

        } catch (WechatApiException e) {
            System.err.println("操作失败：" + e.getErrorCode() + " - " + e.getErrorMsg());
        } finally {
            client.shutdown();
        }
    }
}
```

## 5. 菜单按钮类型详解

### 5.1 点击事件 (click)

用户点击按钮后，微信服务器会推送 `CLICK` 事件到开发者服务器。

```java
new ButtonBuilder()
    .name("点击我")
    .asClick("MY_CLICK_KEY")  // key 值用于事件识别
    .build();
```

### 5.2 跳转网页 (view)

用户点击按钮后，直接跳转到指定的 URL。

```java
new ButtonBuilder()
    .name("访问官网")
    .asView("https://example.com")
    .build();
```

### 5.3 跳转小程序 (miniprogram)

用户点击按钮后，跳转到指定的小程序。

```java
new ButtonBuilder()
    .name("打开小程序")
    .asMiniprogram(
        "wx1234567890abcdef",  // 小程序 AppID
        "pages/index/index",    // 小程序页面路径
        "https://example.com"   // 微信版本低于 6.0.2 时的备用链接
    )
    .build();
```

### 5.4 扫码推事件 (scancode_push)

用户点击按钮后，微信客户端调起扫一扫，扫描结果直接推送给开发者服务器。

```java
new ButtonBuilder()
    .name("扫码")
    .asScanCodePush("SCAN_CODE_KEY")
    .build();
```

### 5.5 下发消息 (media_id)

用户点击按钮后，下发指定的素材消息。

```java
new ButtonBuilder()
    .name("图文消息")
    .asMediaId("MEDIA_ID_123")  // 素材 ID
    .build();
```

### 5.6 二级菜单

一级菜单可以包含最多 5 个二级菜单。

```java
new ButtonBuilder()
    .name("更多功能")
    .addSubButton(new MenuButton.Builder()
        .name("功能 1")
        .asClick("FUNC_1")
        .build())
    .addSubButton(new MenuButton.Builder()
        .name("功能 2")
        .asView("https://example.com/func2")
        .build())
    .build();
```

## 6. 注意事项

### 6.1 菜单数量限制

| 类型 | 限制 |
|------|------|
| 一级菜单 | 最多 3 个 |
| 二级菜单 | 每个一级菜单下最多 5 个 |
| 菜单标题 | 最多 16 个汉字（40 字节） |
| 菜单层级 | 最多 2 级 |

### 6.2 菜单生效时间

- 菜单创建成功后，**24 小时内**生效
- 可关注"公众平台安全助手"公众号，绑定账号后实时预览效果
- 取消关注后，菜单立即失效

### 6.3 调用频率限制

- 每天创建/修改次数有限（通常 5-10 次）
- 删除后重新创建也计入次数
- 建议开发环境使用测试号

### 6.4 权限要求

- 已微信认证的服务号可使用全部菜单功能
- 未认证的服务号/订阅号部分功能受限
- 小程序菜单需要关联小程序

### 6.5 事件推送

配置菜单后，需要在服务器端处理事件推送：
- `CLICK`: 点击推事件
- `VIEW`: 点击链接（不推送）
- `scancode_push`: 扫码推事件
- 等等...

## 7. 错误码参考

|-------|------|---------|
| 40001 | access_token 无效 | 检查 token 是否过期 |

| 40021 | 菜单个数超过限制 | 一级菜单最多 3 个 |
| 40022 | 菜单层级超过限制 | 最多 2 级菜单 |
| 40023 | 不合法的 key 值 | 检查 key 格式 |
| 40024 | 不合法的 url | 检查 URL 格式 |
| 45008 | 权限不足 | 确认账号认证状态 |
| 45029 | 超过日创建次数限制 | 次日再试或申请提升额度 |
